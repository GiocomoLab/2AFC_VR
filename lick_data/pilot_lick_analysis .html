Task is a 2 alternative forced choice paradigm, where the VR cues indicate which of two lickports will be rewarded. Rewards are dispensed regardless of the mouse's behavior at this point. def align_rewards_licks(rf,lf,stepSize=.05):
    interpLicks = interpolate_licks(lf)
    postLicksT = 1.5
    nsteps = int(np.ceil(postLicksT/stepSize))
    lickDat = np.zeros([rf.shape[0],interpLicks.shape[1],nsteps])
    for row in range(rf.shape[0]):
        rewardStartInd = np.abs(interpLicks[:,-1]-rf[row,1])
        firstInd = np.argmin(rewardStartInd)
        rowDat = interpLicks[firstInd:firstInd+nsteps,:].transpose()
        lickDat[row,:,:rowDat.shape[1]] = interpLicks[firstInd:firstInd+nsteps,:].transpose()
        
    return lickDat

def interpolate_licks(lf):
    '''interpolate frame times of licks to get an even grid'''
    stepSize = .05
    rawTime = lf[:,3]
    rawLicksL, rawLicksR = lf[:,0], lf[:,1]
    rawPos = lf[:,2]
    time = np.arange(np.floor(rawTime[0]),np.ceil(rawTime[-1]),stepSize)
    
    interpDat = np.zeros([time.size-1, 4])
    for i in range(time.size-1):
        inds = np.where((rawTime>= time[i]) & (rawTime<=time[i+1]))
        
        if inds[0].size == 0:
            interpDat[i,:] = np.array([0,0,np.nan, time[i]])
        else:
            interpDat[i,:] = np.array([rawLicksL[inds[0]].sum(), rawLicksR[inds[0]].sum(), 
                                        rawPos[inds[0]].mean(), time[i]])
    return interpDat

Clearly mouse 2's tactic is just to lick right and then lick left if the reward doesn't show up. I think this confirms that I need to require the mice actively lick to the side that they think the reward should be.

Mice seem to develop spontaneous anticipitory licking though! This is perhaps due to the explicit reward cue.
